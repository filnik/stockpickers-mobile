# Strings and localisation

Read before adding, changing or deleting a user-visible string.

Two locales: `values/` (English, the default) and `values-it/` (Italian), under
`shared/src/commonMain/composeResources/`. Compose Multiplatform resources, **not**
Android `res/` — which changes two of the rules below.

---

## 1. The rules

1. **Never hardcode a user-visible string** in a ViewModel or a composable. Always
   `stringResource(Res.string.…)`.
2. **Add, change or delete across ALL locales in the same commit.** A key added only
   to `values/` is not a crash — Compose falls back to the default — it is a
   half-Italian screen, which is worse because nothing reports it.
3. **Never leave an English placeholder in a non-English file.** An untranslated
   string is better absent (it falls back) than present and wrong: a placeholder
   looks translated and never gets revisited.
4. **Make the minimum change.** Do not reflow, re-quote or reformat lines you are
   not touching — the diff is the review.
5. **Check for references before deleting.** `grep` the key; a `stringResource` for a
   removed key fails at compile time in CMP, but a key removed from *only* the
   default silently changes behaviour in the other locale.
6. **Preserve formatting**: escapes, format specifiers (`%1$s`, `%d`), and any CDATA
   must survive edits identically in every locale, or the argument order breaks in
   one language only.

---

## 2. Two CMP-specific rules

These do not exist in an Android `res/` project and are easy to get wrong.

### Accessors are generated ONLY from the default locale

`Res.string.foo` exists because `foo` is in `values/strings.xml`. **A key present
only in `values-it/` compiles to nothing** — it is dead weight that no code can
reference and no reader will notice.

So the direction matters: **default first, always.** Add to `values/`, then
translate.

### `translatable="false"` is a convention here, not an enforcement

In an Android project, lint reads that attribute. **The CMP resource generator
ignores it.** Keep using it — it documents intent, and it is free if Android lint is
ever pointed at these files — but do not rely on it to catch anything.

Its practical meaning here is: *this key deliberately has no `values-it/` entry.*
Current examples, all acronyms or field names that are identical in Italian:

```xml
<string name="caption_clenow" translatable="false">clenow</string>
<string name="detail_peg"     translatable="false">PEG</string>
<string name="detail_roic"    translatable="false">ROIC</string>
```

Use it for: symbols (`@`, `•`, `/`), format specifiers alone, URLs and technical
URIs, brand names, ticker/field acronyms, and numeric patterns.

---

## 3. Checking the two locales agree

There is no automated check. The one-liner:

```bash
cd shared/src/commonMain/composeResources
comm -23 <(grep -o 'name="[^"]*"' values/strings.xml   | sort) \
         <(grep -o 'name="[^"]*"' values-it/strings.xml | sort)
```

Everything it prints must be `translatable="false"` in `values/`. Anything else is a
missing translation.

The reverse direction should print nothing at all:

```bash
comm -13 <(grep -o 'name="[^"]*"' values/strings.xml   | sort) \
         <(grep -o 'name="[^"]*"' values-it/strings.xml | sort)
```

Anything here is a key that generates no accessor — dead weight.

## 4. Naming

`area_thing`, lowercase snake_case, grouped by the screen or concern that uses them:

| Prefix | For |
|---|---|
| `leaders_`, `detail_`, `profile_` | screen-specific copy |
| `sort_`, `geo_`, `col_` | control labels and column headers |
| `state_`, `empty_`, `snackbar_` | state and feedback |
| `time_`, `sync_` | relative time and sync status |
| `cd_` | **content descriptions** (accessibility) |
| `action_` | buttons |

`cd_` earns its own prefix because those strings are never seen — only heard — and
they are the first thing to rot.

---

## 5. The build-side gotcha

`:shared`'s composeResources are **not propagated** by the AGP 9 KMP
`androidLibrary` plugin. Two `Copy` tasks in `composeApp/build.gradle.kts` stage
them in the package-qualified layout the runtime expects:

- `stageComposeResForApp` → the APK's assets. Without it a plain `assembleDebug`
  ships **zero** resources and the app crashes on the first `stringResource`.
- `stageComposeResForTest` → the unit-test classpath, so `stringResource` resolves
  in the Roborazzi snapshots.

**Do not "simplify" these away.** Remove them only when the plugin propagates
composeResources to consumers on its own.
